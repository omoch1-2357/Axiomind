1//! Command handler modules for Axiomind CLI.
2//!
3//! This module contains individual handler functions for each CLI subcommand.
4//! Each command is implemented in its own module file with a consistent pattern:
5//!
6//! - Public handler function: `pub fn handle_COMMAND_command(...) -> Result<(), CliError>`
7//! - Module-private helpers: Helper functions specific to that command
8//! - Dependency injection: Output streams (`&mut dyn Write`) passed as parameters
9//! - Error propagation: All errors propagated via `CliError` enum
10//!
11//! # Organization
12//!
13//! Commands are organized by complexity:
14//! - **Simple commands** (Phase 2): cfg, doctor, rng, deal, bench
15//! - **Moderate commands** (Phase 3): play, stats, eval, export
16//! - **Complex commands** (Phase 4): replay, verify, sim, dataset
17//!
18//! # Example
19//!
20//! ```rust,ignore
21//! // Command handlers will be added in Phase 2-4
22//! use axiomind_cli::commands::handle_cfg_command;
23//! use std::io;
24//!
25//! let mut out = io::stdout();
26//! let mut err = io::stderr();
27//! handle_cfg_command(&mut out, &mut err).expect("Command failed");
28//! ```
29
